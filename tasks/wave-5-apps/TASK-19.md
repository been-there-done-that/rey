    return config;
  },

### `apps/web/src/app/` — App Router pages
- `app/layout.tsx` — Root layout with providers
- `app/page.tsx` — Landing/gallery page
- `app/login/page.tsx` — Login page
- `app/upload/page.tsx` — Upload page
- Import WASM module from `src/wasm/` (generated by wasm-pack)
- Connect to Zoo SSE stream using device SseToken

### Upload Resume (key feature)
- Persist `upload_id` in `localStorage` on upload start
- On page load, check `localStorage` for pending upload_id
- Reconnect SSE stream and display stalled upload state
- Prompt user to resume if File System Access API handle is available

### OpenAPI TypeScript Client Generation
- Add to Makefile: `cargo run --bin gen-openapi --manifest-path crates/zoo/Cargo.toml > openapi.json`
- Then: `cd packages/api-client && pnpm openapi-typescript ../../openapi.json -o src/generated.ts`
- Add `utoipa` annotations to all Zoo handlers

### `packages/ui/` — Shared React Component Library
Create shared components (shadcn-based):
- `PhotoGrid` — responsive grid of thumbnails
- `UploadProgressBar` — shows upload progress from SSE events
- `Thumbnail` — renders thumbnail from cache or fetches
- `CollectionList` — lists collections
- Used by both `apps/desktop/src/` and `apps/web/src/`

## Notes
- The web app uses the same UI components as the desktop app (via `packages/ui`).
- WASM is loaded via `asyncWebAssembly` in webpack config.
- The SSE connection uses native `EventSource` (not going through WASM).
- Upload resume uses `localStorage` for persistence and File System Access API for file handles.
- The OpenAPI spec is generated from the Zoo server's `utoipa` annotations.

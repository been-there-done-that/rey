## Objective
Create the Tauri desktop application shell. Thin — no business logic. Imports client-lib and registers commands.

## Files to Create

### `apps/desktop/src-tauri/Cargo.toml`
```toml
[package]
name = "rey-desktop"
version = "0.1.0"
edition = "2021"

[dependencies]
tauri = { version = "2", features = [] }
client-lib = { path = "../../crates/client-lib", features = ["desktop"] }
tokio = { version = "1", features = ["full"] }

[build-dependencies]
tauri-build = { version = "2" }
```

### `apps/desktop/src-tauri/src/main.rs`
```rust
#![cfg_attr(not(debug_assertions), windows_subsystem = "windows")]

use client_lib::AppState;
use common::config::AppConfig;

fn main() {
    let config = AppConfig::from_env();
    let state = AppState::init(config).block_on().expect("Failed to init AppState");

    tauri::Builder::default()
        .manage(state)
        .invoke_handler(tauri::generate_handler![
            client_lib::commands::auth::login,
            client_lib::commands::auth::logout,
            client_lib::commands::auth::register,
            client_lib::commands::auth::get_auth_params,
            client_lib::commands::collections::list_collections,
            client_lib::commands::collections::create_collection,
            client_lib::commands::collections::archive_collection,
            client_lib::commands::files::list_files,
            client_lib::commands::files::get_file,
            client_lib::commands::files::archive_file,
            client_lib::commands::files::download_file,
            client_lib::commands::sync::trigger_sync,
            client_lib::commands::sync::get_sync_status,
            client_lib::commands::upload::upload_file,
            client_lib::commands::upload::cancel_upload,
            client_lib::commands::upload::list_pending_uploads,
            client_lib::commands::thumbnails::get_thumbnail,
            client_lib::commands::thumbnails::evict_thumbnail,
            client_lib::commands::device::register_device,
            client_lib::commands::device::get_device_info,
            client_lib::commands::search::search_files,
            client_lib::commands::search::search_by_date,
            client_lib::commands::search::search_by_location,
        ])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

### `apps/desktop/src-tauri/src/bindings.rs` (build-time codegen)
```rust
use tauri_specta::collect_commands;

fn main() {
    tauri_specta::Builder::<tauri::Wry>::new()
        .commands(collect_commands![
            client_lib::commands::auth::login,
            // ... all commands
        ])
        .export(
            specta_typescript::Typescript::default(),
            "../src/bindings.ts",
        )
        .unwrap();
}
```

## Notes
- The Tauri app is intentionally thin — it's just a shell that imports client-lib.
- `tauri_specta` generates TypeScript bindings at build time.
- The frontend uses the same React components as the web app (via `packages/ui`).
- Platform keychain integration is handled by the `keyring` crate in `local-db`.
